You are Codex Command Line Interface. Build a mock data generation module for a Pension Benefit Guaranty Corporation case workbench suite.

Repository: caseworkbench-mockgen
Language: TypeScript (strict), Node.js 20+
Package manager: pnpm (workspace)
Determinism: all outputs reproducible given the same seed
No personal identifying information: never generate real names, addresses, identifiers, phone numbers, email addresses, or Social Security numbers; use synthetic tokens.

  - GENERATOR_VERSION

  - a SHA256 for every produced artifact
- Do not start a later milestone until current milestone acceptance checks pass.

1) DD.csv (data dictionary) as input

v0.7.4 — Core generator (clean dataset, CSV only)
---------------------------
Engine choice:
- Use @faker-js/faker as the primitive engine (seeded).
- Implement your own small domain generator (do not use mock.js).

Implement:
- generateCleanPopulation(FieldCatalog, nonEmptyFields, GeneratorConfig) -> records[]
- Output includes ONLY fields listed in nonempty.json, in that order.
- Dates output as text yyyy-mm-dd.
- ID and RETSTAT are text.
- Synthetic tokens for identity fields (Cust_ID, BCV_REC_ID, names). Do not generate real names.

Minimum scenarios (configurable with scenarioMix):
- participant_in_pay: ID="1", RETSTAT="1", DOR present, LEV_MB_ARD present (if those fields exist in nonempty list)
- participant_deferred_vested: ID="1", RETSTAT="2", PA_AMB present (if exists)
- participant_active_vested: ID="1", RETSTAT="3", PA_AMB present (if exists)
- participant_not_vested: ID="1", RETSTAT="4"
- beneficiary_in_pay: ID="2", SBCD present, LEV_MB_ARD present (if exist)
- alternate_payee_in_pay: ID="4", LEV_MB_ARD present (if exists)
- de_minimis_lump_sum: ID="1", LS_EST_AMT>0, LS_EST_DATE present (if exist)
- excluded: RETSTAT="5"

Clean invariants (enforce in code when fields exist):
- if DOB and DOH exist: DOB < DOH
- if DOH and DOP exist: DOH <= DOP
- if DOTE exists: DOP <= DOTE
- if ID="1" and RETSTAT="1": DOR present if DOR is in nonempty list
- if ID="2": SBCD present if SBCD is in nonempty list; DOR blank if DOR is in nonempty list
- if DOD and DOR exist and both present: DOR < DOD
- if MSTAT="M": spouse fields present (SFNAME, SLNAME, SDOB) if those fields exist in nonempty list

Output:
- population.clean.csv

Acceptance:
- Determinism test: same inputs + seed => identical CSV bytes and SHA256
- Invariant tests: clean output passes all enforced invariants
